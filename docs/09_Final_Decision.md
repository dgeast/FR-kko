### Q2. API를 사용하지 않는다면 어떤 문제가 있는가?

```
영향도 평가:
╔══════════════════════════════════════════════════╗

║  🟡 날씨/뉴스 미사용: 기능 손실 (중요)          ║
║  🟢 증권 API 미사용: 무시 가능 (선택)           ║

```

#### API별 상세 분석

##### 1️⃣ AI 응답 API (가장 중요)

```
시나리오: ChatGPT/Claude 미사용

❌ 문제점:
  1. 챗봇 기능 전체 불가능
  2. 일반 대화 불가능
  3. 문맥 이해 불가능
  4. 사용자 만족도 거의 0%

⚠️ 기술적 해결책:
  선택지 1: 로컬 LLM (LLaMA 2, Mistral)
  ├─ 비용: $0
  ├─ 응답시간: 5-10초 (API 1초 대비 느림)
  ├─ 품질: 중간 (GPT 대비 50-70%)
  ├─ 리소스: GPU 필수 (비용 증가)
  └─ 개발기간: +2-3주
  
  선택지 2: 규칙 기반 대화 (Rule-based)
  ├─ 비용: $0
  ├─ 응답시간: <1초
  ├─ 품질: 매우 낮음
  ├─ 한계: 정해진 질문만 답변
  └─ 개발기간: 증가 (유지보수 어려움)

✅ 권장 해결책:
  → ChatGPT/Claude API 사용 ($50/월)
  → 성능, 비용, 품질 모두 최적

🎯 결론: API 필수 (비용 저렴하므로 미사용 불필요)
```

##### 2️⃣ 날씨 정보 API

```
시나리오: OpenWeather API 미사용

❌ 문제점:
  1. 날씨 기능 제거 필요
  2. 또는 정적 데이터만 제공
  3. 사용자 경험 저하

✅ 해결책:
  1. OpenWeather 무료 버전 사용 ($0)
     ├─ 60 calls/min 한도 (충분)
     ├─ 실시간 데이터 (정확)
     └─ 개발기간: 없음
  
  2. 다른 무료 API (Weather API)
     └─ 비슷한 기능, 무료
  
  3. 날씨 기능 제거
     └─ 기능 손실

🎯 결론: 무료 API로 충분 (미사용 불필요)
```

##### 3️⃣ 뉴스 정보 API

```
시나리오: NewsAPI 미사용

❌ 문제점:
  1. 뉴스 기능 제거
  2. 또는 RSS 크롤링 필요 (복잡)
  3. 개발복잡도 증가

✅ 해결책:
  1. NewsAPI 무료 버전 ($0)
     ├─ 100 requests/day 한도 (충분)
     ├─ 실시간 뉴스 (정확)
     └─ 개발기간: 없음
  
  2. RSS 크롤링 (대안)
     ├─ 비용: $0
     ├─ 문제: 법적 이슈 가능
     └─ 개발기간: +1주

🎯 결론: 무료 API 권장 (크롤링 피할 것)
```

```

    ↓

    ↓

최적화 (병렬):
Backend ├─────────────┐
        │ Frontend    │
        │ (동시)      │ → 통합
        └─────────────┘
─────────────

구체적 실행:
- Week 1: 환경 구축 (모두)
- Week 2: Backend API 작성 시작
  └─ Frontend: API 목업으로 UI 개발
- Week 3: 양쪽 통합
```

```

구체적 실행:
```

##### 방법 4️⃣: 우선순위 재조정 (절감: 0.5주)

```

1. MVP 명확히 (사용자, 인증, 대화)
2. 자동화 테스트만 (E2E 나중에)
3. 모바일은 Phase 2에서 (선택)
```
