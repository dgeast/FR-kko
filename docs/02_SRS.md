## 2. 기능 요구사항 (Functional Requirements)

### 2.1 사용자 관리 모듈 (User Management Module)

#### FR-UM-001: 사용자 등록 (User Registration)
- **ID:** FR-UM-001
- **제목:** 새로운 사용자 등록
- **설명:** 관리자가 새로운 사용자를 등록하고 얼굴정보를 저장
- **선행 조건:** 
  - 시스템이 초기화되어 있음
  - 현재 등록 사용자 수 < 20명
- **입력:**
  - 사용자 이름/가명 (필수, 1-50자)
  - 이메일 (선택, 유효한 이메일 형식)
  - 카메라 입력 (얼굴 3-5장)
- **처리 로직:**
  1. 사용자 입력 유효성 검사
  2. 카메라에서 얼굴 감지 (3-5회)
  3. 각 얼굴에서 특성 추출 (face encoding)
  4. 특성을 로컬 DB에 저장 (암호화)
  5. 사용자 정보 저장
- **출력:**
  - 성공: "사용자 [이름]이 등록되었습니다" 메시지 + 등록된 사용자 ID
  - 실패: 에러 메시지 (원인 설명)
- **예외 상황:**
  - 중복 이름 등록: "이미 등록된 이름입니다" → 다른 이름으로 재등록 요청
  - 카메라 오류: "카메라를 사용할 수 없습니다" → 재시도 옵션 제공
  - 얼굴 미감지: "얼굴이 감지되지 않았습니다. 다시 시도하세요"
  - 최대 사용자 초과: "최대 20명까지만 등록 가능합니다"
- **비기능 요구사항:**
  - 응답 시간: 10초 이내
  - 데이터 일관성: ACID 보장

#### FR-UM-002: 사용자 조회 (User Retrieve)
- **ID:** FR-UM-002
- **제목:** 등록된 사용자 목록 조회
- **입력:** 없음 (또는 필터 조건)
- **출력:** 등록된 모든 사용자의 목록 (ID, 이름, 등록일시)
- **예외 상황:**
  - 등록된 사용자 없음: "등록된 사용자가 없습니다"

#### FR-UM-003: 사용자 정보 수정 (User Update)
- **ID:** FR-UM-003
- **제목:** 사용자 정보 업데이트
- **입력:**
  - 사용자 ID
  - 수정할 필드 (이름, 이메일, 활성화 상태)
- **출력:** "사용자 정보가 수정되었습니다" 메시지
- **예외 상황:**
  - 사용자 미존재: "해당 사용자를 찾을 수 없습니다"

#### FR-UM-004: 사용자 삭제 (User Delete)
- **ID:** FR-UM-004
- **제목:** 사용자 및 얼굴정보 삭제
- **입력:**
  - 사용자 ID
  - 삭제 확인 (보안)
- **처리 로직:**
  1. 사용자 ID 유효성 검사
  2. 삭제 확인 요청
  3. 사용자 정보 삭제
  4. 연관된 모든 얼굴정보 삭제
  5. 대화 기록 삭제 (선택사항)
- **출력:** "사용자 [이름]이 삭제되었습니다"
- **예외 상황:**
  - 사용자 미존재: "해당 사용자를 찾을 수 없습니다"
  - 삭제 취소: "삭제가 취소되었습니다"

### 2.2 얼굴 인증 모듈 (Face Authentication Module)

#### FR-FA-001: 얼굴 감지 및 인증 (Face Detection & Authentication)
- **ID:** FR-FA-001
- **제목:** 실시간 카메라 입력에서 얼굴 감지 및 등록된 사용자와 비교
- **선행 조건:** 
  - 최소 1명 이상 사용자 등록
  - 카메라 접근 권한
- **입력:** 카메라 비디오 스트림
- **처리 로직:**
  1. 카메라 비디오 프레임 캡처 (30fps)
  2. 각 프레임에서 얼굴 감지 (OpenCV/MediaPipe)
  3. 감지된 얼굴에서 특성 추출 (face encoding)
  4. 추출된 특성을 DB의 모든 등록된 얼굴과 비교
  5. 유사도 점수 계산 (거리값, threshold: 0.6)
  6. 최고 점수 사용자 확인
- **출력:**
  - **인증 성공 (유사도 >= 0.6):**
    ```
    ╔════════════════════╗
    ║ ✅ 인증 성공     ║
    ║ 사용자: [이름]   ║
    ║ 신뢰도: 98%      ║
    ╚════════════════════╝
    ```
    → 대화 모드 시작
  - **인증 실패:**
    ```
    ╔════════════════════╗
    ║ ❌ 미등록 사용자 ║
    ║ 등록 후 사용    ║
    ║ 하시기 바랍니다 ║
    ╚════════════════════╝
    ```
- **예외 상황:**
  - 얼굴 미감지: "얼굴이 감지되지 않습니다. 카메라 앞에 서세요"
  - 여러 얼굴 감지: "한 명씩 인증해주세요"
  - 카메라 오류: "카메라에 접근할 수 없습니다"
- **성능 요구사항:**
  - 응답 시간: < 1초
  - 인식 정확도: >= 95%
  - 오인식률 (False Positive): < 2%

#### FR-FA-002: 얼굴 인증 재시도 (Re-authentication)
- **ID:** FR-FA-002
- **제목:** 인증 실패 시 재시도
- **입력:** 사용자 재시도
- **처리 로직:**
  1. 카메라 상태 확인
  2. 조명 상태 확인
  3. 얼굴 위치 조정 가이드
  4. 재인증 시도
- **재시도 횟수 제한:** 최대 3회
- **최종 실패:** "3회 시도 후 실패했습니다. 관리자에게 문의하세요"

### 2.3 AI 대화 모듈 (AI Chat Module)

#### FR-AC-001: AI 챗봇 시작 (Start Chat Session)
- **ID:** FR-AC-001
- **제목:** 인증된 사용자와 AI 대화 세션 시작
- **선행 조건:** 사용자 인증 완료
- **입력:** 
  - 사용자 ID
  - 선택 주제 (날씨/뉴스/증권/일반)
- **처리 로직:**
  1. 사용자 인증 상태 검증
  2. 대화 세션 생성
  3. 사용자 선호도/맥락 로드
  4. AI 모델 초기화
  5. 인사 메시지 생성
- **출력:**
  ```
  안녕하세요, [사용자이름]님! 👋
  무엇을 도와드릴까요?
  
  주제: [날씨] [뉴스] [증권] [자유대화]
  ```
- **응답 시간:** < 2초

#### FR-AC-002: 사용자 메시지 처리 (Process User Message)
- **ID:** FR-AC-002
- **제목:** 사용자의 메시지를 받아 AI가 응답
- **입력:**
  - 사용자 메시지 (텍스트)
  - 대화 컨텍스트
- **처리 로직:**
  1. 사용자 메시지 유효성 검사
  2. 인터넷 연결 상태 확인
  3. 주제 분류 (NLP)
  4. 필요시 외부 API 호출 (날씨/뉴스/증권)
  5. AI 응답 생성
  6. 응답 내용 검증 (안전성 필터)
  7. 대화 기록 저장
- **출력:**
  - **인터넷 있음:** 최신 정보 포함 응답
  - **인터넷 없음:** 일반 대화만 가능 (오프라인 모드)
- **응답 시간:** < 5초 (API 호출 포함)

#### FR-AC-003: 주제별 대화 모드 (Topic-Specific Chat)
- **ID:** FR-AC-003
- **제목:** 주제(날씨/뉴스/증권)별로 특화된 대화
- **날씨 정보:**
  - API: OpenWeather
  - 정보: 현재 온도, 습도, 강수량, 예보
  - 업데이트 주기: 10분
- **뉴스 정보:**
  - API: NewsAPI, 국내 뉴스 API
  - 정보: 최신 뉴스 헤드라인, 주요 뉴스 요약
  - 카테고리: 정치, 경제, 스포츠, 기술 등
- **증권 정보:**
  - API: Alpha Vantage, 한국거래소 API
  - 정보: 주식 시세, 주요 지수, 변동률
  - 실시간 업데이트

#### FR-AC-004: 대화 기록 저장 (Save Chat History)
- **ID:** FR-AC-004
- **제목:** 모든 사용자-AI 대화 저장
- **저장 항목:**
  - 사용자 ID
  - 메시지 (사용자)
  - 응답 (AI)
  - 타임스탐프
  - 주제
- **저장소:** 로컬 SQLite DB
- **조회:** 사용자는 자신의 대화 기록만 조회 가능

### 2.4 데이터 관리 모듈 (Data Management Module)

#### FR-DM-001: 데이터베이스 초기화 (Database Initialization)
- **ID:** FR-DM-001
- **제목:** 앱 최초 실행 시 DB 생성 및 초기화

  1. SQLite DB 파일 생성
  2. 테이블 스키마 생성
  3. 인덱스 설정
  4. 기본 데이터 삽입

#### FR-DM-002: 데이터 백업 (Data Backup)
- **ID:** FR-DM-002
- **제목:** 정기적인 데이터 백업

#### FR-DM-003: 데이터 내보내기 (Data Export)
- **ID:** FR-DM-003
- **제목:** 데이터를 CSV/JSON 형식으로 내보내기
- **형식:** CSV, JSON
- **포함 항목:** 사용자 정보, 대화 기록

## 3. 비기능 요구사항 (Non-Functional Requirements)

### 3.1 성능 (Performance)

| 요구사항 | 목표값 | 우선순위 |
|---------|-------|---------|
| 얼굴 인식 응답시간 | < 1초 | 높음 |
| AI 응답 시간 | < 5초 | 높음 |
| 사용자 등록 시간 | < 10초 | 중간 |
| DB 쿼리 응답 | < 100ms | 중간 |
| 앱 시작 시간 | < 3초 | 높음 |

### 3.2 보안 (Security)

| 요구사항 | 설명 | 우선순위 |
|---------|------|---------|
| 얼굴정보 암호화 | AES-256 암호화 저장 | 높음 |
| 패스워드 | SHA-256 해싱 + Salt | 높음 |
| API 통신 | HTTPS/TLS 1.3 | 높음 |
| 접근제어 | 관리자/사용자 역할 구분 | 중간 |
| 개인정보보호 | GDPR/개인정보보호법 준수 | 높음 |

### 3.3 가용성 (Availability)

| 요구사항 | 목표값 |
|---------|-------|
| 시스템 가용성 | 99%+ |
| 평균 복구 시간 (MTTR) | < 1시간 |
| 데이터 손실 방지 | RTO < 1일, RPO < 1시간 |

### 3.4 확장성 (Scalability)

- 최대 20명 사용자 동시 처리 가능
- 대화 기록 100만 건 처리 가능
- 디스크 용량: 최대 500MB

### 3.5 호환성 (Compatibility)

| 플랫폼 | OS | 버전 |
|------|----|----|
| 데스크톱 | Windows | 10, 11 |
| 데스크톱 | macOS | 10.15+ |
| 데스크톱 | Linux | Ubuntu 20.04+ |
| 모바일 | iOS | 13+ |
| 모바일 | Android | 8+ |

### 3.6 사용성 (Usability)

- UI는 직관적이고 배우기 쉬워야 함
- 3회 클릭 이내에 주요 기능 접근
- 키보드/터치 둘 다 지원
- 다크 모드 지원

### 3.7 유지보수성 (Maintainability)

- 모듈화된 아키텍처
- 코드 커버리지 >= 80%
- 문서화 완전성 >= 90%
- 기술 부채 관리

User Input → Validation → Face Capture → Face Encoding → DB Storage → Confirmation
