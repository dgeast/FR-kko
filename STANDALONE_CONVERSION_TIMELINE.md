**목표:** 서버-클라이언트 구조 → 각 디바이스 독립 작동  
**분석 날짜:** 2026-05-02

## 📊 현재 구조 vs 목표 구조

### 현재 구조 (서버-클라이언트)
```
PC (서버)
├── 백엔드: FastAPI (8000)
└── 프론트엔드: React (3000)
         ↓ (WiFi)
    모바일/태블릿 (클라이언트)
    ├── 브라우저에서 http://IP:3000 접속
    └── 데이터는 서버에 저장
```

### 목표 구조 (단말기 독립)
```
각 디바이스 독립 작동
├── 샤오미 폰: 자체 앱 설치 → 독립 실행
├── 노트북: 자체 앱 설치 → 독립 실행  
├── 태블릿: 자체 앱 설치 → 독립 실행
└── PC: 자체 앱 설치 → 독립 실행

특징: 서버 불필요, 데이터는 각 디바이스에 로컬 저장
```

## 🛠️ 가능한 기술 선택지

### 옵션 1️⃣: Flutter (권장)
```
장점:
✅ iOS + Android 동시 지원
✅ 빠른 개발 (핫 리로드)
✅ 뛰어난 성능
✅ Google 공식 지원
✅ 하나의 코드베이스

단점:
❌ Dart 언어 학습 필요
❌ 기존 FastAPI 코드 재사용 불가
❌ 빌드 과정 복잡

예상 소요 시간: 7-10일 (완전 개발)
```

### 옵션 2️⃣: React Native
```
장점:
✅ JavaScript (기존 지식 활용)
✅ iOS + Android
✅ 코드 재사용 가능 (일부)

단점:
❌ Flutter보다 느린 성능
❌ 네이티브 의존성 많음
❌ 디버깅 어려움

예상 소요 시간: 8-12일 (완전 개발)
```

### 옵션 3️⃣: Progressive Web App (PWA)
```
장점:
✅ 기존 React 코드 대부분 재사용
✅ 빠른 개발
✅ 오프라인 지원 가능 (서비스 워커)
✅ 모든 OS 지원

단점:
❌ 완전한 네이티브 앱은 아님
❌ 카메라 접근 제한 (브라우저별)
❌ 성능 제한

예상 소요 시간: 2-3일 (수정)
```

### 옵션 4️⃣: Electron (데스크톱 전용)
```
장점:
✅ 기존 React 코드 100% 재사용
✅ 빠른 개발
✅ Node.js 백엔드 포함

단점:
❌ 모바일 불가
❌ 번들 크기 큼 (200MB+)

예상 소요 시간: 1-2일 (모바일 제외)
```

### 옵션 5️⃣: 하이브리드 (Flutter + PWA)
```
개념: 
- 모바일: Flutter 앱
- 웹/테블릿: PWA

예상 소요 시간: 10-15일
```

```
현재:
src/backend/
  ├── face_engine.py (얼굴 인식)
  ├── chat_engine.py (채팅)
  ├── database.py (데이터베이스)
  └── app.py (API)

src/frontend/
  ├── App.js (상태 관리)
  └── components/ (UI)

┌─────────────────────────────────────┐
│ 재사용 가능도 분석                    │
├─────────────────────────────────────┤
│ Flutter: 15-20% (로직만)            │
│ React Native: 30-40% (일부)         │
│ PWA: 80-90% (거의 대부분)           │
│ Electron: 95%+ (완전 재사용)        │
└─────────────────────────────────────┘
```

### 단계별 마이그레이션

#### Step 1: Python 로직 추출 (2일)

□ face_engine.py → 공통 로직

□ database.py → 로컬 저장소 로직

→ 문서화된 인터페이스 정의
```

#### Step 2: 해당 플랫폼 포트 (5-10일)
```
Flutter의 경우:
  - Dart로 로직 재작성
  - google_mlkit 통합
  - Hive (로컬 DB) 사용

PWA의 경우:
  - 기존 React 코드 유지
  - Service Worker 추가
  - IndexedDB 추가

## 📋 결정 기준

### ❓ 어떤 옵션을 선택할까?

**Q1: 모바일 앱(네이티브)이 필수인가?**
- YES → Flutter (8-13일)
- NO → PWA (2-3일)

**Q2: 배포 속도가 중요한가?**
- YES → PWA (2-3일)
- NO → Flutter (8-13일)

**Q3: 높은 성능이 중요한가?**
- YES → Flutter (8-13일)
- NO → PWA (2-3일)

**Q4: 오프라인 지원이 필수인가?**
- YES → Flutter 또는 PWA
- NO → PWA 또는 Electron

**Q5: 개발 팀의 경험도는?**
- React/Web 경험 많음 → PWA 또는 Electron
- Flutter/Dart 경험 있음 → Flutter
- 새로운 학습 원함 → Flutter
